# Docker를 이용한 server 구현, wordpress 동작
과제 시작보다 먼저 걸린 평가 때문에 급히 만든 필기노트...  

### php
- 웹서버 측에서 실행되는 프로그래밍 언어
- HTML을 프로그래밍적으로 생성해주고, 데이터베이스와 상호작용 하면서 데이터 저장 및 표현
- 스크립트 언어라서 html과 호환성이 좋고, 속도가 아주 빨라 많이 사용된다
- 문법이 C와 비슷
- 데이터베이스 (MySQL, Oracle 등) 

### phpmyadmin
- MySQL을 웹상에서 관리할 목적으로 php로 작성된 도구
- MySQL 데이터베이스를 관리하고 데이터를 내보내거나, 여러 서버를 관리하는 등
- Wordpress에서 작성한 글이나 wordpress 환경 등을 관리자 계정처럼 컨트롤 할 수 있었다

### nginx
- 트래픽이 많은 웹사이트를 위한 경량 웹 서버
- 적은 자원으로 효율적인 운용이 가능하기 때문에 Apache보다 자주 쓰는 추세임

#### (임시) 평가하면서 얻은 정보들 
도커를 이용해서 컨테이너를 만들고, 컨테이너에 웹 서버와 서비스를 올려서 도커 파일로 자동화, 설치하는 프로젝트 
도커파일 (저장소의 루트에 존재), srcs (서버 구축에 필요한 파일들) 폴더 필요
1. from debian:buster (os 설치)
2. 오류 메시지를 표시하지 않는 커맨드
3. run 커맨드를 이용해서 필요한 서비스들을 받아
4. run 커맨드를 이용해서 설정을 입혀주고,
5. 서비스를 실행

- docker rm e
- docker rmi test debian -> os 삭제
- docker images -> 도커 이미지를 보여주는 명령어
- docker ps -a -> 컨테이너를 보여주는 명령어 (run 이후에 컨테이너가 생성)
- docker build . -t test -> 도커 파일을 테스트라는 이름으로 빌드
	- 위의 순서대로 (도커파일에 지정한 순서대로) 작업이 수행됨
- docker run -it -p 80:80 -p 443:443 test -> test 이미지를 이용해서 포트를 열고 실행
- localhost:(port) -> 지정한 포트에 연결했을 때 값이 잘 나오는지? (오토인덱스 화면) , http로 잘 리다이렉션 되어야 함
- localhost/wordpress/ 로 들어갔을 때 워드프레스 시작화면이 잘 나오는가?
	- 사이트 제목, 사용자명 등을 지정해주면 워드프레스가 설치되고, 지정한 이메일주소와 비밀번호를 입력하면 블로그가 생성된다!
- SSL/CA 인증서가 있어야 함
- phpmyadmin 서비스는 워드프레스와 연동이 잘 되어야 한다 (계정은 사전에 지정되어 있어야 함)
	- 워드프레스 관리를 할 수 있는 창이 나온다! 신기하다..
