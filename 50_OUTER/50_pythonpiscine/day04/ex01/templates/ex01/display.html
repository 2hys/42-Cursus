Display Process

  • We connect urls of application and project.
  • httpresponse provides inbound http request to application with a text response.
  • render provides rendering html file with appropriate setting and send a request to application.
  • Django uses request and response object to pass state through the system.
  • when a page is requested, Django creates an HttpRequest object that contains metadata about the request
  • and Django loads (returns) the appropriate view with this HttpRequest object.
  • in short, Django receives web requests and return web responses.

How it works

  1. Django Application gets HTTP request from browser.
  2. when received, web application analyzes what request really wants, via POST and GET data
  3. and web application does requested actions (read/write from database, etc...)
  4. after it is done, application returns a response, mostly with showing html page.
